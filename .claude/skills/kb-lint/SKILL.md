# KB Lint

Health-check the knowledge base wiki and fix structural issues.

Follow the **Lint** workflow in `knowledge_base/SCHEMA.md`. That file is the single source of truth for page templates, naming conventions, and workflow steps.

## Guidelines

- Run lint after every batch of ingests or when the wiki feels stale
- Prefer fixing over flagging when the fix is unambiguous
- Keep the report concise — focus on actionable items
